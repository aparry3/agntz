Get your provider API keys

Each link opens the provider's console where you create a key. Paste keys into .env.local at the repo root using the env-var name shown — those are the exact names the SDK reads. Providers marked harness are in the current smoke-test matrix; a missing key just makes that provider SKIPPED.

Quick .env.local template

Drop this in the repo root (./.env.local) and fill in the keys you have. The harness reads it on pnpm --filter @agntz/provider-harness start; any blank/absent key → that provider is skipped.

# --- harness providers ---
OPENAI_API_KEY=
ANTHROPIC_API_KEY=
GOOGLE_GENERATIVE_AI_API_KEY=
MISTRAL_API_KEY=
GROQ_API_KEY=
COHERE_API_KEY=
OPENROUTER_API_KEY=

# --- other SDK-supported providers (optional) ---
XAI_API_KEY=
DEEPSEEK_API_KEY=
PERPLEXITY_API_KEY=
AZURE_OPENAI_API_KEY=
Links current as of May 2026. If any console has moved, the provider's docs/dashboard home will redirect you to the keys page.
